# Qadamha V1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the full Qadamha V1 — a trilingual (EN/Dari/Pashto) static Astro site archiving failed/incomplete Afghan startups, with filterable story index, timeline, OG share cards, contributor docs, and CI.

**Architecture:** Fully static Astro site; stories are schema-validated Markdown files under `src/content/stories/{en,fa,ps}/`, translations linked by shared slug. All pages are generated per-language under `/{lang}/` prefixes with RTL layouts for fa/ps. Client-side filtering runs over a build-generated JSON index. No backend.

**Tech Stack:** Astro 5, TypeScript, zod (via astro:content), @fontsource-variable/vazirmatn, astro-og-canvas (OG images), @astrojs/sitemap, vitest, GitHub Actions, Cloudflare Pages.

### Task 2: i18n core

**Files:** Create `src/i18n/en.json`, `src/i18n/fa.json`, `src/i18n/ps.json`, `src/i18n/index.ts`. Test: `test/i18n.test.ts`.

- [ ] `index.ts` exports:

```ts
export const LOCALES = ['en', 'fa', 'ps'] as const;
export type Locale = (typeof LOCALES)[number];
export const DEFAULT_LOCALE: Locale = 'en';
export const LOCALE_META: Record<Locale, { name: string; nativeName: string; dir: 'ltr' | 'rtl'; htmlLang: string }> = {
  en: { name: 'English', nativeName: 'English', dir: 'ltr', htmlLang: 'en' },
  fa: { name: 'Dari', nativeName: 'دری', dir: 'rtl', htmlLang: 'fa-AF' },
  ps: { name: 'Pashto', nativeName: 'پښتو', dir: 'rtl', htmlLang: 'ps' },
};
export function isLocale(x: string): x is Locale;
export function useTranslations(locale: Locale): (key: string, vars?: Record<string, string | number>) => string; // falls back to en, interpolates {var}
export function localizePath(locale: Locale, path: string): string; // '/{locale}{path}'
```

- [ ] Dictionaries contain identical key sets (nav, home, stories, filters, timeline, story page, contribute, about, footer, statuses, reasons, sectors). Write EN first; FA/PS translated equivalents.
- [ ] Vitest: `useTranslations` returns translated string, falls back to `en` for missing key, interpolates `{count}`; all three dictionaries have identical key sets (structural test — this is the guard that keeps translations in sync forever).
- [ ] Run tests (fail → implement → pass). Commit: `feat: i18n core with EN/Dari/Pashto dictionaries`
